# bmwzgw

Автономный Go-пакет для обнаружения BMW ZGW (Central Gateway) по сети ENET.

Ноль внешних зависимостей — только стандартная библиотека Go.  
Работает на **Windows** и **macOS/Linux** без изменений.

---

## Что делает

Обнаруживает BMW автомобиль, подключённый кабелем ENET к сетевому интерфейсу компьютера:

1. **Этап интерфейса** — как только появляется NIC с адресом `169.254.x.x`, сразу вызывает `onChange` с IP (VIN пустой). Машина видна моментально, до любого UDP-обмена.
2. **Этап ZGW** — отправляет UDP-зонд на `255.255.255.255:6811` и `169.254.255.255:6811`, ждёт ответа с VIN. При получении вызывает `onChange` с заполненными полями `Info`.

Каждые 5 секунд повторяет зондирование. Если ZGW перестаёт отвечать — возвращается к интерфейсному режиму.

### Что возвращает

```
IP       "169.254.138.176"
MAC      "48:C5:8D:90:51:5C"
VIN      "WBA8X51000CF40263"
Model    "G11 740i"
Chassis  "G11"
Platform "S15A"            ← программная платформа ISTA (из кода шасси)
Target   "F020"            ← ECU-адрес из DIAGADR×2 (для внутреннего использования)
Engine   "B58B30M0"        ← код двигателя BMW (из базы типов)
PowerKW  240               ← мощность в кВт
Body     "Sedan"           ← тип кузова
```

**Platform** определяется по коду шасси из встроенной таблицы (~60 шасси). Соответствует идентификатору программной платформы ISTA (VehicleInfo). Примеры:

| Платформа | Автомобили |
|-----------|-----------|
| `F001` | 7 Series F01/F02 |
| `F010` | 5 Series F10/F11, 6 Series F12 |
| `F020` | 1/2/3/4 Series F20–F36 |
| `F025` | X3 F25, X4 F26, X5 F15, X6 F16 |
| `F056` | X1 F48, X2 F39, 2AT F45/F46 (FWD) |
| `S15A` | 5 Series G30, 7 Series G11/G12, X3 G01, X4 G02 |
| `S18A` | 3 Series G20, X5 G05, 8 Series G14–G16, M3/M4 G80–G83 |
| `G045` | iX G45/G46/G48 |
| `G070` | 5 Series G60, 7 Series G70, M5 G84/G90 |

**Target** вычисляется из поля `DIAGADR` в ответе ZGW: `DIAGADR "10"` → `0x10 × 2 = 0x20` → `"F020"`. Нужен для ISTA ECU-адресации; в большинстве задач используйте Platform.

**Model** декодируется из VIN по таблице BMW Baumuster (VIN[3] = тип кузова, VIN[4] = привод, VIN[5] = двигатель), затем уточняется по встроенной базе типов. BMW переиспользует одни и те же буквы VIN[3] между поколениями — disambiguation выполняется по VIN[10] (символ года выпуска), подробнее в разделе «Переиспользование букв VIN[3]» ниже.

**Engine / PowerKW / Body** — заполняются из встроенной базы 4600+ вариантов BMW (файл `bmw_types.csv`, встроен через `//go:embed`). Если автомобиль отсутствует в базе — поля остаются пустыми/нулевыми, остальные поля работают как прежде.

## Использование

### Непрерывный мониторинг (рекомендуется)

```go
package main

import (
    "context"
    "fmt"
    "os/signal"
    "syscall"

    "github.com/atlanteg/bmwzgw"
)

func main() {
    ctx, stop := signal.NotifyContext(context.Background(), syscall.SIGINT, syscall.SIGTERM)
    defer stop()

    bmwzgw.Run(ctx, "", func(info *bmwzgw.Info) {
        // вызывается в фоновой горутине при каждом изменении
        fmt.Println(bmwzgw.FormatBMW(info))
    })
}
```

Вывод:
```
BMW: 169.254.138.176 (no ZGW response)                              ← кабель воткнут, ZGW ещё не ответил
BMW: 169.254.138.176  5UXKS4...  G11 740i  S15A                     ← ZGW ответил
BMW: not found                                                       ← кабель вытащили
```

### Параметр skipIfaceName

Если ваше приложение создаёт виртуальный сетевой адаптер с адресом `169.254.x.x` (VPN, TAP и т.п.), передайте его имя чтобы он не принимался за BMW:

```go
bmwzgw.Run(ctx, "supervpn", func(info *bmwzgw.Info) { ... })
bmwzgw.Run(ctx, "tap0",     func(info *bmwzgw.Info) { ... })
```

### Одиночный запрос (без горутины)

```go
info := bmwzgw.Discover("169.254.138.100") // IP вашего NIC
if info != nil {
    fmt.Printf("VIN=%s  Model=%s  Platform=%s  Engine=%s  %dkW  Body=%s\n",
        info.VIN, info.Model, info.Platform, info.Engine, info.PowerKW, info.Body)
}
```

Ждёт ответа 1.5 секунды, возвращает `nil` если ZGW не ответил.

### Использование полей Info напрямую

```go
bmwzgw.Run(ctx, "", func(info *bmwzgw.Info) {
    if info == nil || info.VIN == "" {
        return // нет машины или VIN ещё не известен
    }
    updateUI(info.IP, info.VIN, info.Model, info.Platform, info.MAC,
             info.Engine, info.PowerKW, info.Body)
})
```

## Структура Info

```go
type Info struct {
    IP       string // 169.254.x.x адрес ZGW
    MAC      string // MAC ZGW, формат "48:C5:8D:90:51:5C"
    VIN      string // VIN 17 символов; "" если ZGW ещё не ответил
    Model    string // напр. "G11 740i"; из VIN + база типов
    Chassis  string // код шасси, напр. "G11"
    Platform string // программная платформа ISTA, напр. "S15A", "S18A", "F020"
    Target   string // ECU-адрес DIAGADR×2, напр. "F020"; для внутреннего использования

    // Поля из базы типов (bmw_types.csv) — пустые/0 если нет в базе:
    Engine  string // код двигателя BMW, напр. "B58B30M0"
    PowerKW int    // мощность в кВт, напр. 240
    Body    string // тип кузова: "Sedan", "Touring", "Coupe", "Sports Activity Vehicle", …
}
```

## База типов BMW (bmw_types.csv)

Встроенный файл `bmw_types.csv` содержит ~4600 вариантов комплектаций, собранных с публичного источника (bimmerarchive.org).

**Колонки:** `family_code`, `model`, `engine`, `power`, `drivetrain`, `chassis`, `steering`, `region`

**Логика подбора варианта (`lookupVariant`):**

1. Фильтр по коду шасси (из VIN[3]).
2. Фильтр по приводу: Rear-Wheel Drive / All Wheel-Drive / Front-Wheel Drive — определяется из VIN[4] и платформы (FWD-платформы: F45, F48, F39 и др.).
3. Фильтр по суффиксу двигателя (напр. `"30d"` из VIN[5]) — ищется в поле `model` базы (без учёта регистра).
4. Если точного совпадения нет — повторный поиск без фильтра по двигателю (только привод).
5. Среди кандидатов: приоритет у `region=Europe` (+2 очка) и `steering=left` (+1 очко).

Для автомобилей, которых нет в базе (напр. G20/G21 — только недавно добавлены), поля `Engine`, `PowerKW`, `Body` остаются пустыми/нулевыми; `Model` и `Chassis` всё равно заполняются из VIN-таблицы.

## ZGW — протокол

**Зонд (6 байт):**
```
00 00 00 00 00 11
```
`0x0011 = 17` — длина VIN. ZGW игнорирует зонды короче 6 байт.

**Ответ ZGW (пример):**
```
00 00 00 32 00 11 DIAGADR10 BMWMAC48C58D90515C BMWVINWBA8X51000CF40263
```

| Поле | Пример | Описание |
|------|--------|----------|
| `DIAGADR` | `10` | hex, адрес ECU / 2 = ISTA target |
| `BMWMAC` | `48C58D90515C` | MAC-адрес ZGW |
| `BMWVIN` | `WBA8X51000CF40263` | VIN, 17 символов ISO 3779 |

**Порт:** `6811 UDP`  
**Источник зонда:** пакет отправляется как с порта `6811` (через постоянный rx-сокет), так и с эфемерных портов, привязанных к каждому `169.254.x.x` интерфейсу, — чтобы гарантировать прохождение через нужный NIC.

## Переиспользование букв VIN[3] между поколениями

BMW присваивает буквы VIN[3] (тип-ключ Baumuster) и переиспользует их в новых поколениях. Это приводит к неоднозначности: одна и та же буква означает разные модели в зависимости от года выпуска.

**Позиция VIN[10]** в BMW кодирует внутренний производственный код, который монотонно растёт со временем и позволяет определить поколение автомобиля. Стандартная схема ISO 3779 (A=2010, B=2011 … R=2024) применима к немецким заводам (WMI `WBA`, `WBS`, `WBY`) и большинству других.

| VIN[10] | Год | VIN[10] | Год |
|---------|-----|---------|-----|
| `A` | 2010 | `K` | 2019 |
| `B` | 2011 | `L` | 2020 |
| `C` | 2012 | `M` | 2021 |
| `D` | 2013 | `N` | 2022 |
| `E` | 2014 | `P` | 2023 |
| `F` | 2015 | `R` | 2024 |
| `G` | 2016 | `S` | 2025 |
| `H` | 2017 | | |
| `J` | 2018 | | |

> `I`, `O`, `Q` не используются (визуально похожи на цифры).

> **Примечание:** BMW South Africa (WMI `X4X`, `WBX`) и некоторые другие заводы используют цифры (`0`–`9`) или нестандартные буквы в VIN[10]. Поскольку цифры (ASCII 48–57) всегда меньше букв (ASCII 65–90), условие `VIN[10] < порогового символа` срабатывает корректно — такие автомобили всегда попадают в ветку F-series.

**Подтверждённые случаи переиспользования** (73 реальных автомобиля, FA XML):

| Ключ | Порог VIN[10] | Шасси (старое/F-series) | Шасси (новое/G-series) |
|------|---------------|------------------------|------------------------|
| `H` | `K` (MY2019) | F48 X1 FWD | G29 Z4 |
| `K` | `G` (MY2016) | F15 X5 / F16 X6 / F85 X5M | G11/G12 7 Series |
| `W` | `N` (MY2022) | F25 X3 | G26 4 Series Gran Coupé |

**Примеры:**
- `WBALX51070C799615` — VIN[3]=`L`, VIN[10]=`C` → **F13** 650i xDrive (не G01).
- `X4XXW394X00P64042` — VIN[3]=`X`, VIN[10]=`0` (BMW SA, 2016) → **F26** X4 xDrive.
- `WBAKS410X00C32596` — VIN[3]=`K`, VIN[10]=`0` (digit < `G`) → **F15** X5 xDrive30d.
- `WBA7C41070G762791` — VIN[3]=`7`, VIN[10]=`G` → **G11** 730d xDrive.
- `WBAHF51090WX29248` — VIN[3]=`H`, VIN[10]=`W` (>= `K`) → **G29** Z4 M40i.
- `WBAHS120205F03712` — VIN[3]=`H`, VIN[10]=`5` (digit < `K`) → **F48** X1 sDrive18i FWD.

Таблицы `gseriesIntroMY` и `fseriesAltKeys` в `zgw.go` содержат все известные случаи. При обнаружении нового VIN с неверным декодированием — проверьте VIN[3] и VIN[10] и добавьте запись в обе таблицы.

## Требования

- Go 1.21+
